WORKSHOP GIT

Ítem 1: clonando repositorio

Vamos a comenzar clonando el repositorio de este proyecto. Para ello, ingresamos al siguiente link:

https://github.com/CeciliaBGH/ejercicio_2

Se emplea el comando git clone 'url_del_repositorio'

Ítem 2: creamos una rama y trabajamos en ella

Creamos una rama para realizar cambios en ella, sin afectar las características del proyecto principal. Empleamos el comando git chechout -b 'nombre_de_la_rama'.

El flag -b indica que se creará una nueva rama y el comando chechout especifica que nos cambiaremos a la nueva rama.

Ítem 3: uso del archivo gitignore

Creamos un archivo de prueba, por ejemplo: prueba.txt.

A continuación incorporamos el mismo al archivo .gitignore.

Posteriormente, subimos los cambios locales a remoto. ¿Se visualiza el archivo prueba.txt en el repositorio de GitHub?

Ítem 4: modificamos el código en la rama creada

Realizar algún cambio en el código de la rama. ¿Impacta en el proyecto principal? ¿Cómo podemos hacer que impacte en el proyecto principal?

Ítem 5: fusionamos la rama creada a la rama main

En primer lugar, debemos posicionarnos en la rama que quedará luego de la fusión. En nuestro caso, nos movemos a la rama main: git checkout main

A continuación, fusionamos el contenido de la rama secundaria hacia la rama main. Usamos el comando git merge rama-secundaria

Luego de la fusión, ¿se visualizan en el proyecto los cambios que se habían realizado en la rama secundaria?

Ítem 6: registro del historial

Se ponen a prueba los comandos log, relativos al registro de historial
